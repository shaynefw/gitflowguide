Git Flow Tutorial

This tutorial provides a simple guide to the best practices and commands for using Git in a professional workflow.

1. Create a New Branch

Start by creating a new branch for your feature or bugfix:

git checkout -b feature/my-new-feature

2. Stage Your Changes

After making changes to your files, stage them for commit:

git add .

3. Commit Your Changes

Commit your changes with a descriptive message following conventional commit guidelines: 

git commit -m "feat: add new feature

- Implement new functionality
- Update documentation
- Fix related bugs"

4. Switch to the Main Branch

Switch to the main branch to pull the latest changes:

git checkout main

5. Pull the Latest Changes

Ensure your main branch is up to date:

git pull origin main

6. Merge Main into Your Branch

Switch back to your feature branch and merge the latest changes from main: 

git checkout feature/my-new-feature
git merge main

7. Push Your Feature Branch

Push your feature branch to the remote repository:

git push origin feature/my-new-feature

8. Create a Pull Request

Navigate to your Git hosting service (GitHub, GitLab, etc.) and create a pull request from your feature branch to the main branch.

Moving Changes from Main to a Feature Branch

If you forgot to switch branches before making changes, you can move the changes to a new branch: 

git stash
git checkout -b feature/my-new-feature
git stash pop

Then, stage and commit your changes:

git add .
git commit -m "feat: add new feature

- Implement new functionality
- Update documentation
- Fix related bugs"
